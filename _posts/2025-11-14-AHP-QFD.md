## 1. 層級分析法（AHP, Analytic Hierarchy Process）

### **1.1 目的**

AHP 是一種多準則決策方法，透過層級結構與成對比較，協助決策者在多重因素下做出客觀權重分配與選擇。

### **1.2 特點**

* 將複雜問題拆解成層級結構（目標 → 準則 → 子準則 → 替選方案）
* 使用成對比較矩陣（Pairwise Comparison）
* 透過特徵向量求得各因素權重
* 提供一致性檢定（Consistency Ratio, CR）

### **1.3 使用步驟**

1. **建立層級結構模型**

   * 明確定義決策目標、準則與替選方案。
2. **成對比較（Pairwise Comparison）**

   * 以 Saaty 1–9 標度比較各因素相對重要性。
3. **計算權重**

   * 透過矩陣歸一化或特徵向量法求得權重。
4. **一致性檢定**

   * 若 CR < 0.1 視為可接受。
5. **整合權重並作決策**

### **1.4 優點與缺點**

**優點：**

* 結構清晰，有助於系統化思考
* 可量化主觀判斷
* 適合多準則決策問題

**缺點：**

* 成對比較在準則過多時負擔大
* 主觀性仍無法完全避免
* 一致性檢定未通過需重新評估

## 2. 品質機能展開（QFD, Quality Function Deployment）

### **2.1 目的**

QFD 用於將顧客需求轉換為產品或服務的設計規格，確保設計過程符合顧客期望。

### **2.2 核心工具：品質屋（House of Quality, HOQ）**

品質屋是 QFD 最具代表性的矩陣，用於連結：

* 顧客需求（WHATs）
* 技術需求（HOWs）
* 關聯矩陣（顧客需求與技術需求的關係強弱）
* 技術重要度評估
* 基準分析（Benchmarking）

### **2.3 使用步驟**

1. **蒐集顧客需求（VOC, Voice of Customer）**
2. **將顧客需求量化與排序**
3. **定義技術規格（HOWs）**
4. **建立關聯矩陣（WHATs–HOWs）**
5. **競品比較與目標設定**
6. **計算技術特性的重要度**
7. **延伸至後續設計、製程、品質管控**

### **2.4 優點與缺點**

**優點：**

* 將顧客需求直接導入設計決策
* 能降低開發風險，提高顧客滿意度
* 提供跨部門整合的架構

**缺點：**

* 建置品質屋耗時，需要大量資料
* 結果仰賴團隊經驗與判斷
* 若需求整理不精確，後續設計會偏離目標

## 3. AHP 與 QFD 的比較

| 項目       | AHP             | QFD                |
| -------- | --------------- | ------------------ |
| **目的**   | 多準則決策與權重分析      | 將顧客需求轉換為設計規格       |
| **主要工具** | 成對比較矩陣          | 品質屋（HOQ）           |
| **資料來源** | 專家判斷較多          | 顧客需求（VOC）為主，搭配團隊判斷 |
| **適用階段** | 決策評估階段          | 產品/服務設計與開發階段       |
| **輸出**   | 各準則或方案之權重       | 技術需求的重要度與設計導引      |
| **優點**   | 清晰的層級結構、可整合主觀判斷 | 顧客導向、利於跨部門協作       |
| **缺點**   | 主觀性較高、比較矩陣複雜    | 建置耗時、資訊需求量大        |

---

## 4. AHP 與 QFD 整合應用

AHP 也常被用來：

* 評估 QFD 中顧客需求的重要度
* 評估 HOWs 的相對重要性

整合流程：

1. 使用 AHP 計算顧客需求（WHATs）之權重
2. 將權重帶入 QFD 的品質屋
3. 計算技術需求（HOWs）的加權重要度
4. 作為後續產品設計與優化的依據
